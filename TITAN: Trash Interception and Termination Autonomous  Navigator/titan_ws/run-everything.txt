# TITAN Robot - Operations Guide

This guide provides detailed instructions for operating the **TITAN** (Trash Interception and Collection Autonomous Navigator) robot.

## 1. System Check (Before Every Run)

### Correct Workspace
Ensure you are using the new TITAN workspace:
```bash
cd ~/titan_ws
source install/setup.bash
```
*(Tip: Add `source ~/titan_ws/install/setup.bash` to your `~/.bashrc` to auto-load it).*

### Power & Connectivity
- **Check Power Stability:** 
  Run `vcgencmd get_throttled` on the Pi.
  - `throttled=0x0`: Power is stable.
  - `throttled=0x50005`: Under-voltage detected (check battery/cable).

- **Check Network:**
  To edit Wi-Fi settings: `sudo nano /etc/netplan/50-cloud-init.yaml` (or similar).

- **Check Hardware Connections:**
  The robot uses persistent names for devices. Verify they exist:
  ```bash
  ls -l /dev/arduino  # Should point to /dev/ttyUSBx
  ls -l /dev/lidar    # Should point to /dev/ttyUSBx
  ```

## 2. Modes of Operation

### A. Mapping (Creating a New Map)
Use this mode to drive the robot around and build a map of the environment.

**On Robot (SSH):**
```bash
ros2 launch titan_bringup mapping.launch.py
```
*Note: You may see "queue full" warnings at startup. This is normal. Wait for "Registering sensor".*

**On Laptop/Remote (for Control & Visualization):**
1. **Control:**
   ```bash
   ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -p speed:=0.2 -p turn:=0.8
   ```
2. **Visualize:**
   Launch `rviz2` and add "Map" (topic `/map`) and "LaserScan" (topic `/scan`).

**Saving the Map:**
When finished, save the map **before** stopping the mapping node:
```bash
ros2 run nav2_map_server map_saver_cli -f ~/titan_ws/src/titan_bringup/maps/campus_map
```

### B. Navigation (Autonomous Driving)
Use this mode to make the robot drive itself within an existing map.

**On Robot (SSH):**
```bash
ros2 launch titan_bringup navigation.launch.py
```
*This loads `maps/campus_map.yaml` by default.*

**On Laptop/Remote (RViz):**
1. Set **2D Pose Estimate** to align the robot's position with the map.
2. Set **Nav2 Goal** to send the robot to a destination.

### C. Hardware Bringup (No SLAM/Nav)
Use this to test sensors or run custom code without the navigation stack.
```bash
ros2 launch titan_bringup bringup.launch.py
```
*Publishes: `/scan`, `/odom`, `/tf`, `/cmd_vel`.*

## 3. Maintenance & Development

### Rebuilding the Workspace
If you change any code, you must rebuild:
```bash
cd ~/titan_ws
rm -rf build/ install/ log/   # Optional: Clean clean build
colcon build --symlink-install
source install/setup.bash
```

### Common Issues
- **"Queue full" warning:** Normal at startup. If it persists during operation, the Arduino connection might be unstable.
- **Lidar not spinning:** Check battery voltage and `/dev/lidar` permission.
- **Drifting Odom:** Check if wheels are slipping or if `ticks_per_meter` needs tuning in `arduino_bridge_node.py`.
